# X-ModalProof

implementation for:

**X-ModalProof: Real-Time Explainable Ownership Verification for Multimodal and Edge-Deployed AI Models**

## Status

This repository currently implements:

- deterministic config + seed pipeline
- text-mode watermark training and verification loop
- signature construction + threshold selection + cosine verification
- smoke/debug/full run modes
- reproducibility logs and assumptions tracking

Image, multimodal, explainability, attacks, and deployment modules are scaffolded and ready for iterative extension.

## Frozen paper-results mode (artifact regeneration)

The repository includes a frozen reference file at `results/paper_results.json`.

- Scripts in `scripts/` regenerate artifacts from this file.
- They **do not** rerun full paper experiments from scratch.
- Missing paper-reported values remain empty/null by design.

Use:

```bash
python scripts/make_tables.py --paper-results results/paper_results.json
python scripts/make_figures.py --paper-results results/paper_results.json
python scripts/run_baselines.py --paper-results results/paper_results.json
python scripts/run_attacks.py --paper-results results/paper_results.json
python scripts/run_explainability.py --paper-results results/paper_results.json
python scripts/run_threshold_sensitivity.py --paper-results results/paper_results.json
python scripts/run_trigger_size_ablation.py --paper-results results/paper_results.json
python scripts/benchmark_latency.py --paper-results results/paper_results.json
python scripts/export_onnx.py --paper-results results/paper_results.json
```

See `docs/results_reference.md` for policy and provenance details.



## Mode summary

- **frozen/reference mode**: regenerate artifacts from `results/paper_results.json` only.
- **scaffold mode**: dependency-light synthetic smoke/debug/full pipeline for CI and local validation.
- **real_full mode**: local optional-dependency paths using user-prepared processed datasets/triggers and local run outputs.





### Real-full DistilBERT + AG News (local, optional dependencies)

Install optional full-real dependencies (not used by CI):

```bash
pip install -r requirements-full.txt
```

Prepare local files referenced by `data/manifests/datasets_manifest.yaml`:
- `data/processed/text/ag_news/train.csv`
- `data/processed/text/ag_news/validation.csv`
- `data/processed/text/ag_news/test.csv`
- `data/processed/triggers/text/ag_news_triggers.csv`

Then run:

```bash
python scripts/train.py --config configs/full_real_distilbert_agnews.example.yaml
python scripts/eval.py --config configs/full_real_distilbert_agnews.example.yaml --run-dir outputs/full_real/text_distilbert_agnews_seed0
```

If dependencies are missing, the scripts fail clearly with an install hint.

### Real-full attacks / robustness evaluation (implemented scope in this stage)

Run from existing real_full run directories:

```bash
python scripts/run_attacks.py --mode real_full --attack pruning --run-dir outputs/full_real/text_distilbert_agnews_seed0 --strength 0.3
python scripts/run_attacks.py --mode real_full --attack finetuning --run-dir outputs/full_real/image_mobilenetv2_cifar10_seed0 --finetune-epochs 1
python scripts/run_attacks.py --mode real_full --attack distillation --run-dir outputs/full_real/text_distilbert_agnews_seed0 --distill-epochs 1
```

Outputs are written to `outputs/full_real/attacks/<attack>/<run_name>/` and include:
- `attack_metrics.json`
- `attack_metrics.csv`
- `attack_log.txt`
- `config_snapshot.yaml`

Current support boundary:
- DistilBERT+AG News: pruning, finetuning, distillation implemented.
- MobileNetV2+CIFAR-10: pruning, finetuning, distillation implemented.
- CLIP/ViLT verification-only paths: model-mutation attacks are currently unsupported in this stage and fail clearly.

### Real-full latency benchmarking (implemented scope in this stage)

Run local latency benchmarking from real_full run directories:

```bash
python scripts/benchmark_latency.py --mode real_full --run-dir outputs/full_real/text_distilbert_agnews_seed0 --backend pytorch --hardware local_cpu
python scripts/benchmark_latency.py --mode real_full --run-dir outputs/full_real/image_mobilenetv2_cifar10_seed0 --backend onnxruntime --hardware local_cpu
python scripts/benchmark_latency.py --mode real_full --run-dir outputs/full_real/image_mobilenetv2_cifar10_seed0 --backend tensorrt --hardware jetson_nano
```

Outputs are written under `outputs/full_real/deployment/latency/`:
- `<run_name>_<backend>_latency.json`
- `<run_name>_<backend>_latency.csv`
- `latency_log.txt`

Support boundary in this stage:
- `pytorch`: implemented for DistilBERT+AG News and MobileNetV2+CIFAR-10.
- `onnxruntime`: implemented when ONNX model exists and `onnxruntime` is installed.
- `tensorrt`: requires local TensorRT runtime + engine artifacts; unavailable environments fail clearly.

Important: latency benchmarking is local-only; CI does not measure real edge hardware latency.


### Real-full artifact aggregation from validated local outputs (implemented scope in this stage)

Artifact-generation scripts now support local real-output aggregation:

```bash
python scripts/make_tables.py --mode real_full --outputs-root outputs/full_real
python scripts/make_figures.py --mode real_full --outputs-root outputs/full_real
```

Optional controls:
- `--output-dir` (default `outputs/full_real/artifacts`)
- `--strict` (fail if required sources are missing)
- `--include-missing` (emit explicit missing/null rows instead of skipping)
- `--source-mode` (label written into source tracking metadata)

Generated outputs:
- `outputs/full_real/artifacts/tables/`
- `outputs/full_real/artifacts/figures/`
- `outputs/full_real/artifacts/artifact_generation_log.txt`
- `outputs/full_real/artifacts/artifact_sources.json`

Policy in `--mode real_full`:
- No silent fallback to `results/paper_results.json`.
- Missing sources are represented as missing/null (unless `--strict`).
- Generated artifacts are explicitly labeled with source mode.
- `results/paper_results.json` is never overwritten by artifact generation.

```bash
python scripts/run_explainability.py --mode real_full --method shap --run-dir outputs/full_real/text_distilbert_agnews_seed0
python scripts/run_explainability.py --mode real_full --method gradcam --run-dir outputs/full_real/image_mobilenetv2_cifar10_seed0

```

Notes:
- Outputs are **supportive evidence**.
- They are **not causal proof** and **not legal/forensic proof**.
- `captum_ig` is accepted as an explicit alias for text attribution and writes outputs under `outputs/full_real/explainability/captum_ig/`.
- Routing in `--mode real_full` now uses `config_snapshot.yaml` (dataset/model names) before run-directory naming fallbacks.
- CLIP attention rollout currently uses a documented alternative token-ablation summary because stable attention maps are not exposed in the selected backend wrapper.

## Real paper-level reproduction

This repository currently provides:
- **Frozen/reference mode**: reads `results/paper_results.json` and regenerates artifacts.
- **Scaffold mode**: lightweight synthetic validation for CI/local checks.
- **Real full mode contract**: validates user-prepared local datasets/outputs and fails clearly if missing.

Important:
- Current scaffold full mode is executable for code-path validation DistilBERT/MobileNetV2/CLIP/ViLT production.
- GitHub Actions is intended only for smoke/scaffold tests.
- Users must prepare datasets locally under `data/raw/` and `data/processed/`.
- Users must place real experiment outputs under `outputs/full_real/`.

Validate local full inputs first:

```bash
python scripts/validate_full_inputs.py --manifest data/manifests/datasets_manifest.yaml --check-all
```

Then run scaffold/full commands:

```bash
python scripts/train.py --config configs/full.yaml
python scripts/eval.py --config configs/debug.yaml --run-dir outputs/debug_text_b4_seed0
```

Warning:
Do **not** interpret artifact-regenerated tables/figures as newly rerun experiments unless they are generated from validated real local outputs.
